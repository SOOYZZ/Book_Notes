# 2.1 理解词嵌入 — 学习导读

## 一、为什么需要词嵌入?

先抛出**核心问题**:神经网络只会算数学(矩阵乘法、求梯度),但文本是离散的符号(字、词、标点)。"猫"和"狗"对计算机来说就是两串编码,没有任何数学意义。

所以我们必须把文本**翻译成数字**——不是随便编个号(那样"猫=1, 狗=2"毫无语义),而是要把每个词变成一个**连续值的向量**,让"语义相近的词,向量也相近"。

这个翻译过程就叫 **embedding(嵌入)**。

---

## 二、嵌入是什么?一句话定义

> **嵌入的本质:把离散对象(单词、图像、文档)映射到一个连续向量空间中的点。**

不只是文本,视频、音频都可以嵌入,只是每种数据要用**不同的嵌入模型**——文本的嵌入模型不能拿去嵌入音频。

想象一张二维平面图(类似书中图 2-3):

```
           ↑ 维度 2
           |
   ●eagle  ●sparrow
   ●hawk   ●robin             ← 鸟类聚集在左上
           |
           |     ●Tokyo
           |     ●Paris        ← 城市聚集在中间
           |     ●Berlin
           |
           |              ●Japan
           |              ●France    ← 国家聚集在右下
           |              ●Germany
           +---------------------→ 维度 1
```

注意两个规律:

1. **同类词聚成一团**:鹰、麻雀、知更鸟挨得近
2. **类间距 vs 类内距**:不同种类的鸟之间的距离,通常比"国家 ↔ 城市"的距离更近

这就是 word2vec 这类算法学出来的效果。它的核心思想叫**分布假说**:

> 出现在相似上下文中的词,往往含义相似。

比如"我吃了一个\_\_\_"后面接"苹果""香蕉""橘子"的概率都很高,模型就会把这三个词的向量学得很接近。

---

## 三、几个关键概念

### 1. 维度(dimension)

向量有多长,就是多少维。

| 模型                     | 嵌入维度 |
| ------------------------ | -------- |
| 二维可视化示例           | 2        |
| 小型 GPT-2 (1.17 亿参数) | 768      |
| 最大 GPT-3 (1750 亿参数) | 12,288   |

**权衡**:维度越高,能表达的语义细微差别越多,但计算越慢、内存越大。GPT-3 用 12,288 维,意味着每个词被表示为一个长度为 12,288 的浮点数列表。

> ⚠ 高维向量没法可视化(我们的感官最多到三维)。书里的二维图只是教学示意。

### 2. 词嵌入 vs 句子/段落嵌入

| 嵌入粒度      | 主要用途                                   |
| ------------- | ------------------------------------------ |
| **词嵌入**    | 本书重点。类 GPT 模型逐词生成,所以用词嵌入 |
| 句子/段落嵌入 | RAG(检索增强生成)常用,本书不涉及           |

### 3. 用预训练嵌入 vs 自己训练嵌入

这是本节最重要的一个判断:

- **用 word2vec 等预训练嵌入**:拿来即用,但嵌入和你的任务无关
- **大语言模型自己训练嵌入**:嵌入层是模型输入层的一部分,跟着模型一起训练更新

**LLM 的选择是后者**。原因:嵌入可以针对**特定任务和数据**优化。比如训练一个医学 LLM,"阳性"这个词在医学语料里会被嵌入到和"癌症""感染"很近的位置,而通用 word2vec 嵌入做不到这种领域定制。

> 📌 第 3 章还会提到:LLM 不仅有**输入嵌入**,还会生成**与上下文相关的输出嵌入**。同一个词"bank"在"river bank"和"bank account"里输出嵌入是不同的,这是 Transformer 注意力机制带来的。

2. word2vec 学习嵌入的核心思想是什么?用一句话说。
3. 为什么 GPT 不直接用 word2vec 预训练好的嵌入?
4. 嵌入维度越高越好吗?

<details>
<summary>参考答案(自己想完再展开)</summary>

1. 因为整数 ID 之间的数值关系(1 < 2 < 3)和词的语义没关系。"猫=1, 狗=2, 苹果=3" 会让模型误以为"狗"在"猫"和"苹果"中间。嵌入用向量,让语义近的词向量也近。
2. 出现在相似上下文中的词,具有相似的含义(分布假说)。
3. 因为 LLM 把嵌入层作为自己的一部分一起训练,嵌入可以针对具体任务和数据优化,效果优于通用的 word2vec。
4. 不是。高维度能捕捉更细的语义关系,但计算成本上升,需要权衡。
