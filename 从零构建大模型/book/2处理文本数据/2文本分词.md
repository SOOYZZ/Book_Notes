# 2.2 文本分词

## 一、目标

把这段输入文本:

```
"I HAD always thought Jack Gisburn rather a cheap genius..."
```

切成这样一个词元列表:

```python
['I', 'HAD', 'always', 'thought', 'Jack', 'Gisburn', 'rather', 'a', 'cheap', 'genius', '--', ...]
```

注意几个关键点:

- 标点符号(`,` `.` `--` `?`)要**独立成项**,不能和单词粘在一起
- **不转小写**——大写形式能帮模型区分专有名词(Jack)和普通名词、理解句子结构
- 空白字符暂时**丢掉**(简化输出;实际场景按需保留)

---

## 二、分词器的演进:三个版本

作者用一个很好的教学路径——**从最朴素的版本开始,逐步发现问题、修复问题**。

### 版本 1:只按空白字符切

```python
import re
text = "Hello, world. This, is a test."
result = re.split(r'(\s)', text)
# ['Hello,', ' ', 'world.', ' ', 'This,', ' ', 'is', ' ', 'a', ' ', 'test.']
```

**问题**:`Hello,` 把单词和逗号粘在一起了。

### 版本 2:按空白 + 逗号 + 句号切

```python
result = re.split(r'([,.]|\s)', text)
# ['Hello', ',', '', ' ', 'world', '.', '', ' ', 'This', ',', '', ' ', 'is', ' ', 'a', ' ', 'test', '.', '']
```

标点分开了,但出现了空字符串 `''` 和空格 `' '`。用列表推导式清理:

```python
result = [item for item in result if item.strip()]
# ['Hello', ',', 'world', '.', 'This', ',', 'is', 'a', 'test', '.']
```

**问题**:碰到 `?` `"` `--` 这些还是处理不了。

### 版本 3(最终版):处理各种标点

```python
result = re.split(r'([,.:;?_!"()\']|--|\s)', text)
result = [item.strip() for item in result if item.strip()]
```

正则解释一下,这部分:

| 部分             | 含义                                                                      |
| ---------------- | ------------------------------------------------------------------------- |
| `[,.:;?_!"()\']` | 字符类:逗号、句号、冒号、分号、问号、下划线、感叹号、双引号、括号、单引号 |
| `\|--`           | 或者两个连字符 `--`(老式英语小说中表示破折号)                             |
| `\|\s`           | 或者任何空白字符                                                          |
| 最外层 `(...)`   | 捕获组——`re.split` 会**保留**捕获的分隔符,所以标点不会被丢掉              |

---

## 三、关键技巧:`re.split` 的捕获组

这是这一节最容易绊住人的细节。看个对比:

```python
re.split(r'\s', "a b c")        # ['a', 'b', 'c']        ← 分隔符被丢掉
re.split(r'(\s)', "a b c")      # ['a', ' ', 'b', ' ', 'c']  ← 分隔符被保留
```

**给正则加括号 = 把分隔符也留在结果里**。这正是我们想要的——标点本身就是有用的词元,不能丢。

## 五、一个值得注意的设计决定:大小写和空白

书里提到两个**不显眼但重要**的选择:

### 1. 为什么不全部转小写?

朴素的想法:`Jack` 和 `jack` 是同一个词,统一变小写省事。

但作者明确**不这么做**。原因:

- **专有名词识别**:`Jack`(人名) vs `jack`(千斤顶) 是不同的词
- **句子结构**:句首大写帮助模型理解句子边界
- **生成时的大小写**:模型最终也要会**生成**大写——句首字母、人名首字母大写是英文写作规范。如果训练时全转小写,模型永远学不会大写

### 2. 空白字符:丢还是留?

| 选择               | 适用场景                                                     |
| ------------------ | ------------------------------------------------------------ |
| **丢掉**(本节做法) | 普通自然语言,省内存省算力                                    |
| **保留**           | 对结构敏感的文本,比如 Python 代码——缩进有语法意义!丢掉就废了 |

**GPT 用的 BPE 分词器实际上保留空白**(在 2.5 节会看到),书里这里先简化一下让你入门。

## 七、自测

1. 为什么正则表达式用 `(\s)` 而不是 `\s`?去掉括号会发生什么?
2. 为什么不把所有字符转成小写?给两个理由。
3. 列表推导式 `[item for item in result if item.strip()]` 在过滤什么?为什么用 `.strip()` 而不是直接 `if item`?

<details>
<summary>参考答案</summary>

1. 加括号是**捕获组**,`re.split` 会保留捕获的分隔符。不加括号,空白和标点会被直接丢掉——但我们要标点作为词元。

2. ① 区分专有名词与普通名词(Jack vs jack);② 模型要学会**生成**大小写(句首、人名等),训练时丢了大写,生成时就回不来。

3. 过滤掉空字符串 `''` 和**纯空白字符串** `' '` `'\t'` `'\n'`。直接 `if item` 只能过滤掉 `''`,纯空格 `' '` 会被当成真值保留下来。`.strip()` 删除两端空白后再判断是否为空,两种情况一并过滤。

</details>
